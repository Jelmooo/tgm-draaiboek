0 Eenmalig instellen (alleen de eerste keer ooit)

Wie ben je? (voor je commits)

git config --global user.name  "Jouw Naam"

Inloggen bij GitHub (met de GitHub CLI 'gh')

gh auth login

Kies bij de vragen: GitHub.com → HTTPS → Yes → Login with a web browser. Je krijgt een code die je in je browser plakt.

1 Een nieuw project online zetten

Open een terminal in je projectmap (in Windows Verkenner: klik in de adresbalk, typ powershell, Enter) en typ:

git init -b main                  # maak hier een git-project
git add .                         # alle bestanden klaarzetten
git commit -m "Eerste versie"     # eerste opslagpunt
gh repo create <projectnaam> --private --source=. --push
Privé of openbaar? --private = alleen jij ziet het. Wil je dat iedereen het mag zien, gebruik dan --public.

Klaar! Je code staat nu op https://github.com/<jouw-naam>/<projectnaam>.

1 De dagelijkse routine

Telkens als je iets hebt veranderd en wilt vastleggen, typ je deze vier regels:

git status                      # wat heb ik veranderd?
git add .                       # alle wijzigingen klaarzetten
git commit -m "korte uitleg"    # opslagpunt maken (op je pc)
git push                        # naar GitHub sturen (nu online)

Voorbeeld:

git add .
git commit -m "Tekst op homepagina aangepast"
git push

2 Wat doet elk commando?

CommandoWat het doet
git status Laat zien wat er veranderd is. Verandert niets — gebruik het zo vaak je wilt.
git add . Neem alle wijzigingen mee in het volgende opslagpunt. (De punt = alles.)
git commit -m "…" Maak een opslagpunt met een kort briefje erbij. Blijft nog op je eigen pc.
git push Stuur je opslagpunten naar GitHub. Pas nu staat het online.
git pull Haal de nieuwste versie op vanaf GitHub (handig op een 2e pc of server).
git log --oneline Toon de lijst met al je opslagpunten (commits).
⚠️ De meest gemaakte beginnersfout commit en push zijn twee aparte stappen. commit = opslaan op je pc (werkt ook offline). push = naar GitHub sturen. Niets staat online totdat je git push hebt gedaan.

? Handige extra's & problemen oplossen

SituatieOplossing
Is alles veilig opgeslagen én online? git status → "working tree clean" + "up to date with origin/main" = veilig online.
Niet-opgeslagen wijzigingen weggooien git restore . (let op: maakt niet-opgeslagen wijzigingen ongedaan!)
"Please tell me who you are" bij commit Stel je naam/e-mail in (zie tab ① → Eenmalig instellen).
push vraagt steeds om wachtwoord Niet je GitHub-wachtwoord gebruiken. Log in met gh auth login.

CasaOSdraait als Docker

Je haalt de code op de server op en bouwt er een Docker-container van. Dit gaat ervan uit dat je project een docker-compose.yml bevat.

1 Eerste keer installeren

Ga naar je CasaOS-server (via SSH of de ingebouwde terminal-app) en typ:

# 1. Code ophalen vanaf GitHub (in de map met de live websites)
cd "/Live Websites"
git clone <repo-url>
cd <projectnaam>

# 2. Docker-container bouwen en starten
docker compose up -d --build

Testen kan via http://<ip-van-je-casaos>:<poort> (de poort staat in docker-compose.yml). De nette publieke URL wordt geregeld door Cloudflare, dat naar diezelfde poort verwijst.

📁 Waar staat wat op de server? Code/website: /Live Websites/<projectnaam>  ·  Data van de app: /DATA/APPS DATABASE/<projectnaam> (ingesteld als volume in docker-compose.yml).
OnderdeelBetekenis
git clone …Haalt het hele project van GitHub naar de server.
docker compose upBouwt de app tot een Docker-container en start 'm.
-d"detached": draait op de achtergrond, ook na uitloggen.
--buildBouwt de container (opnieuw) op basis van de nieuwste code.
🔒 Privé-repo? Dan vraagt git clone om in te loggen. Gebruik niet je GitHub-wachtwoord, maar log eenmalig in met gh auth login op de server — of zet de repo op publiek.

⚙ Handige Docker-commando's

Voer deze uit in de projectmap op de server (/Live Websites/<projectnaam>):

CommandoWat het doet
docker compose psDraait de app? Toont de status.
docker compose logs -fLive meekijken met de app (Ctrl+C om te stoppen).
docker compose restartApp herstarten zonder opnieuw te bouwen.
docker compose downApp stoppen en de container verwijderen.

i Een update uitrollen in 2 stappen

Heb je iets aangepast en wil je dat op de server zien? Dat gaat altijd zo:

A
Op je pcpush naar GitHub
→
B
Op CasaOSpull + opnieuw bouwen

A Op je pc — wijziging online zetten

git add .
git commit -m "wat je veranderd hebt"
git push

B Op de CasaOS-server — ophalen + opnieuw bouwen

cd "/Live Websites/<projectnaam>"
git pull                        # nieuwste code ophalen
docker compose up -d --build    # container opnieuw bouwen + starten
📦 Je gegevens blijven bewaard De data staat in een apart volume onder /DATA/APPS DATABASE/<projectnaam>, los van de code. Een update gooit je gegevens dus niet weg.
Bij twijfel: begin altijd met git status. Dat commando verandert nooit iets.
